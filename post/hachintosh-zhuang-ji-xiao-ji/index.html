Hachintosh装机小记

2021-03-04 4 min read # Hachintosh

Stay hungry, stay foolish.

   去年年底的时候，乘着年终奖壮胆，就折腾了一下黑苹果，美其名曰能感受苹果系统，还能闲暇时候来一把Dota，其实还是太穷给闹的，直接买白苹果的各位，我要真心说一句，我想和你们做朋友。甚至为了省下200块钱的系统安装费，各种查资料翻帖子，一番折腾下来，不仅认识了Github这种宝藏网站，也初步了解了什么是EFI，什么是OC，什么叫Hachintoch...最终，成功安装好了系统之后还是挺有成就感的。生命在于折腾，赶紧自我安慰一下。

其实是心疼钱包

  本着经济但是不失13格的原则，无性价比配置如下：

配件 型号
主板 华擎B460M-ITX/ac
CPU i5 10400
内存 英睿达8G DDR4 2666*2
硬盘 海康卫视C2000 pro 1T
显卡 iGUP
电源 全汉MS450
机箱 Ghost s1复刻版
无线网卡 DW1820A
风扇 ID 40X

系统

MacOS Big Sur 11.1+ Win10

引导概况

  • OpenCore version:0.6.5 release
  • 需要用工具GenSMBIOS 生成新的三码，在/EFI/OC/config.plist 文件中进行修改,否则登录apple ID会出现问题
  <dict>
    <key>SystemSerialNumber</key>
    <string>W00000000001</string>
</dict>

<dict>
    <key>MLB</key>
    <string>M0000000000000001</string>
</dict>

<dict>
    <key>SystemUUID</key>
    <string>00000000-0000-0000-0000-000000000000</string>
</dict>
  • 安装MacOS之前，BIOS需要关闭WAN功能。当系统安装成功后, 在config.plist的NVRAM-- Add-- 7C436110-AB2A-4BBB-A880-FE41995C9F82--boot-args中加入brcmfx-drive=0，然后重启进入BIOS开启WAN以实现蓝牙和wifi功能。（如果直接使用本EFI，尽量先去掉引导参数brcmfx-drive=0，等系统安装成功后再添加）
    （注：brcmfx-drive=0这个参数我是在系统安装成功后加入的，没有加入这个引导参数的话，打开WAN功能后就无法启动了。不打开WAN功能的时候，wifi和蓝牙就无法正常使用。所以无线网卡尽量选择其他的热门博通免驱网卡，相对来说DW1820A还是要稍微麻烦一点）
    EFI文件请移步github：https://github.com/EVOHISO/Hackintosh-Asrock-B460M-ITX-ac-i5-10400

ESP设置

  因为是但硬盘安装双系统，网上的EFI设置教程不知道为什么都不成功，最后在B站黑苹果屋的视频留言区找到了解决方案。
  将win10的EFI文件夹中的Microsoft文件夹直接拷贝到本MacOS的EFI中，并将EFI文件夹放入ESP分区即可，据说是win10系统会自动寻找Boot文件夹。（有大神知道的话，请指教）

功能

正常功能

  • 随航、接力、夜览
  • wifi与蓝牙（原装intel网卡已更换为博通DW1820A）
  • 有线网卡
  • 休眠(休眠后可以正常唤醒）
  • USB定制
  • 声卡定制

不正常功能

  • Hackintool中显示为coffee lake平台（暂时并不影响使用）
  • 显示器亮度调节（后续会使用MonitorControl）
  • Openshell
